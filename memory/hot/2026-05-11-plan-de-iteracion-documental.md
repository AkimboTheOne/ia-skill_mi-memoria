# Plan de iteración documental

## Memoria

Regla operativa para convertir la estabilización documental en una secuencia reutilizable de mejoras.

## Resumen

Primero contrato, luego uso, después ideas, al final versión.

## Desarrollo

- Mantener `prompt` para activación conversacional.
- Mantener `bash` para comandos técnicos.
- Separar usuario, gobernanza y memoria curada.
- Expandir docs de usuario antes de tocar el README maestro.
- Registrar cada cambio importante en `CHANGELOG.md`.
